# 作业
1. 使用 redis benchmark 工具, 测试 10 20 50 100 200 1k 5k 字节 value 大小，redis get set 性能。
2. 写入一定量的 kv 数据, 根据数据大小 1w-50w 自己评估, 结合写入前后的 info memory 信息 , 分析上述不同 value 大小下，平均每个 key 的占用内存空间。

## 环境 
windows WSL，redis: docker image
```
docker run -p 6379:6379 --name redis -d redis
```
## 参考
https://redis.io/docs/reference/optimization/benchmarks/

## 1.
命令：
```
redis-benchmark -t get,set -n 100000 -d {字节大小} -q
```
```shell
$ redis-benchmark -t get,set -n 100000 -d 10 -q
SET: 30826.14 requests per second
GET: 25900.03 requests per second

$ redis-benchmark -t get,set -n 100000 -d 20 -q
SET: 16398.82 requests per second
GET: 11964.58 requests per second

$ redis-benchmark -t get,set -n 100000 -d 100 -q
SET: 27886.22 requests per second
GET: 19704.43 requests per second

$ redis-benchmark -t get,set -n 100000 -d 50 -q
SET: 20903.01 requests per second
GET: 15475.08 requests per second

$ redis-benchmark -t get,set -n 100000 -d 200 -q
SET: 25940.34 requests per second
GET: 14465.50 requests per second

$ redis-benchmark -t get,set -n 100000 -d 1000 -q
SET: 14102.38 requests per second
GET: 19098.55 requests per second

$ redis-benchmark -t get,set -n 100000 -d 5000 -q
SET: 11431.18 requests per second
GET: 14847.81 requests per second
```
结论：字节大小对读写性能影响不大。
